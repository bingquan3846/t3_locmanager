T3 LocManager Help

Export

Allows you to export localisable data from the database tables (pages, pages_language_overlay, tt_content) so that you can translate it with an external translation tool such as SDL TagEditor or PASSOLO. Export is done in an XML format that can be easily imported into most CAT-Tools or software localization tools. After translation you can reimport the localised file and localised pages will be automatically generated and content inserted. See Import for more information.

Start page ID

Indicates the ID of the page where T3 LocManager should start exporting.

Recursion

By checking the Export recursively option T3 LocManager will export all pages and content elements under the selected starting point.

Tidy

HTML Tidy is an open source program and library for checking and generating clean XHTML/HTML. If the Tidy option is checked T3 LocManager will check content from TYPO3 for X/HTML validity and allow for XML compatible export.

Ignore hidden

The user is given the possibility to ignore hidden pages and hidden content elements while exporting. So hidden elements (mostly elements still under development) will not be translated.

Ignore deleted

Deleted pages and content elements will never be exported.

Ignore localized

When exporting T3 LocManager checks if the pages or content elements to be exported have already been translated. If it detects already localized elements T3 LocManager stops giving the warning that some elements are already localized. The user then has the option to uncheck the option ignore localized ....

Overwrite

Various overwrite options can be set, e.g. to overwrite a localization package, when exporting an updated version.

Localizable page types

Are the localizable page types. Different types can be selected to adapt the export and translation process to the users' needs.

Localizable content elements

Are the localizable content element types. Different types can be selected to adapt the export and translation process to the users' needs.

Localizable fields

Are the localizable database fields. Different fields can be selected to adapt the export and translation process to the users' needs.

PASSOLO *.xfg

T3 LocManager allows for automatic generation of a PASSOLO *.xfg file that contains the required XML rules. The rules are dynamically build according to the options defined by the user. The *.xfg file can be directly imported into PASSOLO. If you want to include the PASSOLO *.xfg file in the localization package check PASSOLO *.xfg from the General tab in the export options.

SDL TRADOS *.ini

T3 LocManager allows for automatic generation of a SDL TRADOS *.ini file that contains the required tag settings for the translation. The *.ini file is dynamically build according to the options defined by the user. If you want to include the SDL TRADOS *.ini file in the localization package check SDL TRADOS *.ini from the General tab in the export options. The *.ini file will then be added to the *.zip file.

Import

Allows for import of translated files that have been previously exported with T3 LocManager. On import new localized pages will be automatically inserted into the database tables. Various options and automatic checks allow user-defined import. See options set status to hidden, overwrite, and remove.
